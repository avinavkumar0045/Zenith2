[2026-06-23] PHASE 2A: LOCATION INTELLIGENCE ENGINE
===================================================

8. Core Location Subsystem
- Designed `LocationIntelligenceObject` as the single canonical type for location data.
- Built `useLocationStore` in Zustand to manage `activeLocation` and persist `recentLocations` (max 10) to localStorage.
- Established a global `EventBus` using `mitt` for decoupled module communication.

9. Providers & Services
- Integrated OpenStreetMap's Nominatim API via `NominatimProvider` and abstracted it through `GeocodingService`.
- Integrated `suncalc` via `DayNightService` to accurately calculate Sunrise, Sunset, Twilight, and Day/Night status based on coordinates and current time.
- Built the `LocationService` orchestrator that ties coordinates to geocoding, day/night parsing, and automatically dispatches to the Store and Event Bus.
- Added `requestBrowserLocation` capabilities with `sessionStorage` guard for first-time visits.

10. Globe Interaction & Camera Sync
- Wired Cesium's `ScreenSpaceEventHandler` in `GlobeService` to emit raw clicks.
- Intercepted click events in `LocationService` to reverse-geocode and trigger system updates.
- Refactored `LocationLayer` to purely listen to `locationChanged` events, draw markers via `MarkerService`, and automatically pilot the `CameraService`.

11. User Interface Components
- `LocationSearch`: Real-time Nominatim search with a beautiful frosted glass dropdown.
- `LocationStatus`: Elegant pill-shaped top-bar displaying Name, Day State, and Local Time.
- `LocationCard`: Extensible bottom-left panel showcasing detailed geospatial insights (Lat/Lon, Sunrise, Sunset).
- `RecentLocations`: Quick switcher for previous places.
- Synced state with URL query parameters `?lat=xx&lon=yy` allowing instant deep-linking.
